## チェックリスト

必要な項目だけを選んで使う想定。

## 1. UI一貫性

- タイポグラフィ
  - 見出し/本文/注釈の階層が一貫している
  - 行間と文字サイズの組み合わせが読みやすい
  - 数字や英字が混ざっても崩れない
- 余白とレイアウト
  - 余白が一定のスケール（例: 4/8/12/16...）で構成されている
  - セクション間の間隔ルールが統一されている
  - グリッドや整列が揃っている
- 色
  - 意味色（成功/警告/危険）が一貫している
  - コントラストが不足していない
  - 状態（pressed/focus/disabled）の表現がある
- コンポーネント
  - ボタンの種類（primary/secondary等）が定義されている
  - 入力欄のラベル/ヘルプ/エラーの配置が統一されている
  - リスト、カード、モーダルの構造が揃っている
- アイコン
  - 線/塗りのスタイルが混在していない
  - サイズと整列が揃っている
  - 意味が曖昧なアイコンにラベルがある

## 2. UXヒューリスティック（要点）

- 状態の可視化
  - 処理中、完了、失敗が分かる
- 実世界との整合
  - 用語と順序が自然
- ユーザー主導
  - 戻る、キャンセル、やり直しができる
- 一貫性と標準
  - 同じ操作は同じ結果
- エラー防止
  - そもそも間違えにくい
- 想起負荷の低減
  - 覚えなくても見れば分かる（候補、例示、既定値）
- 最小限のデザイン
  - 重要情報が埋もれない
- エラー回復
  - 次の行動が分かる
- ヘルプ
  - 必要な場所に短い説明がある

## 3. 状態設計

- loading
  - 待ちの理由が分かる
  - 連打防止がある
- empty
  - 理由と次の行動がある
- error
  - 行動に変換された文言
  - リトライや代替導線
- success
  - 完了と次の行動
- offline/poor network
  - 再試行と最低限表示

## 4. Webアクセシビリティ

- キーボード操作
  - フォーカス移動が成立
  - フォーカスが視認できる
  - モーダルでフォーカストラップ
- セマンティクス
  - 見出し、フォームラベルが適切
- コントラスト
  - テキストが読める
  - UIの境界が見える
- フォーム
  - エラーが関連付く
- 動き
  - 過剰アニメーションがない

## 5. モバイルUX

- タップ領域
  - 小さすぎない
  - 隣接が密で誤タップしない
- 片手操作
  - 主要操作が届く
- 戻る
  - 一貫している
- 入力
  - 適切なキーボード
  - ラベルが消えない
- Safe Area
  - 被らない
- 読み上げ
  - ボタン名が意味として読める

## 6. マイクロコピー

- 具体的
- 行動につながる
- ユーザーを責めない
- 用語が一貫している
- ラベルは短く、説明は必要な時だけ
