## Chrome DevTools MCP 運用メモ

このファイルは「Web UIを根拠付きで監査する」ための手順を固定化する。
ツール名や引数は環境で差が出ることがあるため、実環境のMCPスキーマを正とする。

## 使いどころ

- DOM構造とアクセシビリティ構造を観察する
- 実際の表示と状態（disabled、aria、フォーカス）を裏取りする
- Console/NetworkでUI不具合の根拠を取る
- 体感速度が悪い原因の当たりをつける

## 基本方針

- スナップショット取得を優先する
- 必要な箇所だけスクリーンショットで証跡を残す
- 操作は副作用がない範囲を基本にする

## よく使う操作の流れ

- 対象ページを開く（新規タブまたは遷移）
- スナップショットを取る
  - 主要導線（CTA、ナビ、フォーム）を要素として特定する
- レスポンシブ確認
  - 幅を変えて、崩れ、隠れ、到達性を確認する
- エラー裏取り
  - Consoleメッセージを確認する
  - Networkで失敗や遅いリソースを確認する
- 体感速度（必要な場合）
  - トレースを取り、重い処理の当たりをつける

## 監査で見るポイント

- クリック可能な要素がボタンとして認識されるか
- フォーカス移動で主要操作に到達できるか
- disabled状態が見た目と意味で一致しているか
- エラー時の表示がユーザー行動に変換されているか
- ネットワーク失敗時の回復導線があるか

## レスポンシブ確認（定型）

- 代表的な幅で確認する
  - デスクトップ
  - タブレット相当
  - モバイル相当
- 観察点
  - 主要CTAがファーストビューから消えていないか
  - 固定ヘッダやフッタがコンテンツを隠していないか
  - 文字の折返しやはみ出し
  - ホバー依存のUIがモバイルで詰んでいないか

## 裏取りのログ化

レポートには以下の形で根拠を残す
- どの画面で何を観察したか
- 重要箇所の要素情報（ラベル、ロール、状態）
- 必要箇所のスクリーンショット（位置が分かる説明付き）
- Console/Networkの要点（エラー内容、失敗率、重いリソース）
