## Mobile MCP 運用メモ

このファイルは「モバイルUIを根拠付きで監査する」ための手順を固定化する。
ツール名や引数は環境で差が出ることがあるため、実環境のMCPスキーマを正とする。

## 使いどころ

- 画面に出ている要素をアクセシビリティ要素として観察する
- タップ領域、配置、ラベル、状態を裏取りする
- 入力体験（キーボード、バリデーション、エラー回復）を観察する
- 回転やSafe Areaの崩れを確認する

## 基本方針

- 画面要素リストの取得を優先する
- どうしても要素が取れない場合のみ座標操作で補う
- 副作用がある操作（購入、送信、削除）は許可前提

## 観察の最小手順

- 端末を特定する
- アプリを起動する
- 各ステップで以下をセットで取得する
  - スクリーンショット
  - 画面要素リスト（アクセシビリティ要素）
- 指摘には、要素のラベル、ロール、位置情報を根拠として書く

## 重要な観察ポイント

- どこを押せばよいかが分かるか
- 押した後のフィードバックがあるか
- 戻るが一貫しているか
- 入力補助があるか（キーボード種別、補完、フォーマット）
- エラーから回復できるか（どこを直すか分かる、入力保持）
- Safe Areaと固定要素の干渉がないか

## 画面回転

- 縦横を切り替えて以下を確認する
  - 主要CTAが消えない
  - 表示が切れない
  - 入力状態が壊れない
