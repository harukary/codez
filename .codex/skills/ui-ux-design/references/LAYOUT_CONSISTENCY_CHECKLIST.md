# 画面間一貫性チェックリスト（ui-ux-design 用）

「同じ役割は同じ型」を作るためのチェックリストです。
各項目でズレがあれば、どれが正で、なぜそうするか（原則）までセットで扱います。

## 情報の骨格

- 画面タイトル/見出しの階層が統一されているか
- 説明文（補足）が入る位置と粒度が揃っているか
- 主要 CTA の位置・優先順位が揃っているか

## ナビゲーションと戻り

- 戻る/キャンセル/閉じるの扱いが統一されているか
- モーダル/フルスクリーン/別画面の使い分けが説明できるか

## 一覧→詳細（代表パターン）

- 一覧のスキャン性（行/カード/サムネ）と情報粒度が揃っているか
- 詳細での主要アクション配置が揃っているか
- 一覧と詳細の情報の重複/欠落が不自然になっていないか

## フォーム（代表パターン）

- 入力ラベル、補足、エラー表示、必須表記の型が揃っているか
- 入力の分割（段階 vs 一括）の方針があるか
- 保存/更新/破棄の導線が一貫しているか

## 状態設計

- 空状態（0件）で「次に何をすべきか」が統一されているか
- ローディング（骨格/スピナー/インライン）の方針があるか
- エラー（復帰手段/再試行/問い合わせ）の型が揃っているか

## コピー/命名

- 同じ概念に同じ言葉を使っているか（用語ブレがないか）
- 破壊的操作の文言や確認の型が揃っているか

## プラットフォーム差分（必要な差分だけを許容）

- Web/iOS/Android で変える理由が説明できる差分か
- 変える場合も「役割」と「期待結果」が同じに保たれているか
